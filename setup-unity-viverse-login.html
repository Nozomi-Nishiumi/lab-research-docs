クイックスタート — ビルド済みサンプルで動かす

設定済みログインサンプルのインポート

手動で構築する代わりに、完全なログイン UI とシーンを含むパッケージをそのまま使う

⬇ Login_Documentation_Sample.unitypackage

重要:インポート順序 ── 依存関係の欠落を避けるため、必ずこの順序でインポートする。

  1. VIVERSE SDK Unity パッケージ(SDK_v0.96.unitypackage)をインポート:Assets → Import Package → Custom Package… → SDK パッケージを選択 → Import。
  2. TextMeshPro をインポート(プロジェクトに未導入の場合):Window → TextMeshPro → Import TMP Essential Resources。
  3. ログインサンプルパッケージをインポート:Assets → Import Package → Custom Package… → Login_Documentation_Sample.unitypackage を選択 → Import。
確認 ── 3つのパッケージをすべてインポートしたら、Assets/Scenes/ から Login_Leaderboard_Documentation_Sample シーンを開くと、完全なログインサンプルの動作を確認できます。ログイン UI を手動でステップごとに構築したい場合は、以下の STEP 01 から進めてください。
STEP 01

VIVERSE パッケージのインポート

  1. Unity で新規プロジェクトを作成する。
  2. Assets → Import Package → Custom Package… で VIVERSE unity パッケージを選択し、Import をクリック。
  3. 追加されたスクリプトを確認:LoginManager.cs・CloudSaveService.cs・HttpServer.cs・ViverseSDK 1.jslib などが利用可能になる。コードの編集は不要。
STEP 02

2カラムのログイン UI を作成

左カラムに操作(Login / Clear Data ボタン)、右カラムに情報(ステータス・アカウント・トークン)を置く構成です。操作と情報を分離することで、認証状態をひと目で把握でき、ログイン開始やリセットの操作も分かりやすくなります。

Canvas のセットアップ

  1. GameObject → UI → Canvas(名前を LoginCanvas に)。
  2. 自動生成される EventSystem はシーンに残しておく。
  3. LoginCanvas の設定:
    • Render Mode: Screen Space - Overlay
    • Canvas Scaler → UI Scale Mode: Scale With Screen Size、Reference Resolution: 1920×1080

中央パネルの作成

  1. LoginCanvas を右クリック → UI → Panel(LoginPanel にリネーム)。
  2. Rect Transform で中央のアンカープリセットを選択。
  3. Width ≈ 1000、Height ≈ 280、Pos X = 0、Pos Y = 0 に設定。

左カラム(ボタン)

  1. LoginPanel を右クリック → Create Empty(ButtonColumn にリネーム)。
  2. ButtonColumn に Vertical Layout Group を追加:
    • Padding: すべてゼロ(デフォルトのまま)
    • Spacing: 20
    • Child Alignment: Upper Center
    • Child Force Expand の Width/Height を無効化
  3. RectTransform のアンカープリセットを middle left に設定(アンカーマトリクスで Alt+左)。
  4. ButtonColumn を右クリック → UI → Button (TextMeshPro)(LoginButton にリネーム)。Width = 180、Height = 40。子のテキストを「Login」、フォントサイズ約24、色をダークグレー(#333333)に。
  5. LoginButton を複製して ClearDataButton にリネームし、ラベルを「Clear Data」に変更。
  6. (任意)ボタンにフィットするようカラムを縮めたい場合は、Content Size Fitter(Horizontal/Vertical Fit = Preferred Size)を追加。

右カラム(情報グループ)

  1. LoginPanel を右クリック → Create Empty(InfoGroup にリネーム)。
  2. InfoGroup に Vertical Layout Group を追加:
    • Padding: デフォルトのゼロのまま(全辺0)
    • Spacing: 18
    • Child Alignment: Upper Left
  3. アンカープリセットを middle right に設定(Alt+右)。Width ≈ 500、Height ≈ 103。
  4. InfoGroup を右クリック → UI → Text - TextMeshPro(StatusText にリネーム):テキスト「Status: Ready」、フォントサイズ約20、色 #000000、Alignment を Upper Left、Width ≈ 500 / Height ≈ 30。
  5. StatusText を2回複製:1つ目 → AccountText(テキスト「Account: 」)、2つ目 → TokenText(テキスト「Token: 」)。各複製とも Width ≈ 500 / Height ≈ 30 を維持。
  6. (任意)InfoGroup に Content Size Fitter を追加。

カラムの配置

左右のカラムをパネルの端から対称に配置する

ButtonColumn(左)InfoGroup(右)

Pivot(0, 0.5)(1, 0.5)

STEP 03

コントローラースクリプトの追加

LoginUIController を LoginPanel に追加

RequireComponent 属性により、Unity は LoginManager も自動で追加する

C# · LoginUIController.cs
using UnityEngine;
using UnityEngine.UI;
using TMPro;
using ViverseSDK.Login;   // LoginManager を公開する名前空間をインポート

/// <summary>
/// VIVERSE ログインの UI レイヤーを担当。
/// LoginManager のイベントを購読し、ボタン/ラベルを更新する。
/// </summary>
[RequireComponent(typeof(LoginManager))]
public class LoginUIController : MonoBehaviour
{
    [Header("App Settings")]
    [SerializeField]
    private string appId;         // VIVERSE App ID(Inspector で設定)

    [Header("UI References")]
    [SerializeField] private Button loginButton;
    [SerializeField] private Button clearDataButton;
    [SerializeField] private TMP_Text statusText;
    [SerializeField] private TMP_Text accountText;
    [SerializeField] private TMP_Text tokenText;

    private LoginManager loginManager;

    private void Awake()
    {
        // LoginManager コンポーネントを取得し、ボタンのハンドラーを設定
        loginManager = GetComponent<LoginManager>();

        loginButton.onClick.AddListener(() => loginManager.StartLogin());
        clearDataButton.onClick.AddListener(() =>
        {
            loginManager.ClearLoginData();
            UpdateStatus("Tokens cleared");
        });

        // LoginManager のイベントを購読し、UI を認証状態と同期させる
        loginManager.OnStatusUpdated += UpdateStatus;
        loginManager.OnTokenUpdated += UpdateToken;
        loginManager.OnUserInfoUpdated += UpdateAccount;
        loginManager.OnLoginStateChanged += ToggleLoginButton;
    }

    private void Start()
    {
        // 初期状態のステータスを即時表示
        UpdateStatus("Ready");

        // Inspector で指定した App ID を使ってログインフローを初期化
        loginManager.Initialize(appId);
    }

    private void OnDestroy()
    {
        // リークや古い参照を避けるため、必ずイベント購読を解除する
        loginManager.OnStatusUpdated -= UpdateStatus;
        loginManager.OnTokenUpdated -= UpdateToken;
        loginManager.OnUserInfoUpdated -= UpdateAccount;
        loginManager.OnLoginStateChanged -= ToggleLoginButton;
    }

    /// <summary>ステータスラベルを更新し、コンソールにログ出力する。</summary>
    private void UpdateStatus(string message)
    {
        if (statusText) statusText.text = message;
        Debug.Log("[LoginUI] " + message);
    }

    /// <summary>トークン全体を出力しないよう、切り詰めたアクセストークンを表示する。</summary>
    private void UpdateToken(string token)
    {
        if (!tokenText) return;

        tokenText.text = string.IsNullOrEmpty(token)
            ? "Token: <empty>"
            : $"Token: {token.Substring(0, Mathf.Min(20, token.Length))}...";
    }

    /// <summary>ログイン結果から返されたアカウント ID を表示する。</summary>
    private void UpdateAccount(string accountId)
    {
        if (!accountText) return;

        accountText.text = string.IsNullOrEmpty(accountId)
            ? "Account: <none>"
            : $"Account: {accountId}";
    }

    /// <summary>セッションが有効になったらログインボタンを無効化する。</summary>
    private void ToggleLoginButton(bool isLoggedIn)
    {
        if (loginButton) loginButton.interactable = !isLoggedIn;
    }
}

STEP 04

HttpServer の追加(Editor / Windows テスト用)

HttpServer コンポーネントを追加

LoginPanel を選択した状態で、パッケージに含まれる HttpServer コンポーネントを追加する(ローカルリダイレクトフローのため、LoginManager が自動的に参照する)

HttpServer が必要な理由 — 認証フローの仕組み:

  1. ブラウザリダイレクトフロー ── Editor/Windows モードで Login をクリックすると、LoginManager がデフォルトブラウザを開き、VIVERSE ログインページへ遷移する。
  2. コールバック処理 ── 認証成功後、VIVERSE はブラウザを認証トークン付きで localhost の URL(http://localhost:40078/)へリダイレクトする。
  3. ローカルサーバー ── HttpServer はポート 40078 でローカル HTTP サーバーを起動してこのリダイレクトを待ち受け、認証コールバックデータ(アクセストークンとアカウント ID)を受け取る。
  4. データ処理 ── サーバーはコールバックを処理して認証情報を抽出し、Unity アプリケーションからアクセスできるよう PlayerPrefs に保存する。
  5. Unity との連携 ── 保存された認証情報は、LoginManager が Unity 内でログイン処理を完了するために使用される。
注 ── このコンポーネントは Editor と Windows ビルドでのみ必要。WebGL ビルドでは認証は JavaScript ブリッジ(.jslib ファイル)が直接処理するため、HttpServer は不要です。
STEP 05

WebGL ブリッジ(WebGL ビルドの場合のみ)

Assets/Plugins/WebGL/ViverseSDK 1.jslib(パッケージ付属)が存在することを確認します。手動での設定は不要。LoginManager が WebGL を検出し、自動的にブリッジを呼び出します。

STEP 06

ログインフローのテスト

Editor / Windows でのテスト

  1. Play を押す。
  2. Login をクリック → ブラウザで VIVERSE ログインを完了。
  3. ステータス/アカウント/トークンのラベルが更新されることを確認。データは PlayerPrefs に永続化される。

WebGL でのテスト

  1. WebGL プレイヤーをビルドしてホスト(ローカルサーバーまたは VIVERSE)。
  2. Login をクリック → .jslib が SSO フローを処理し、成功するとラベルが更新される。

APPENDIX

ログインサービスのデータ契約

LoginManager はログイン結果を以下にデシリアライズします:

C# · データ契約
Serializable class Result {
    public string access_token;
    public string account_id;
    public int    expires_in;
    public string state;
}
キャッシュとクリア ── トークンとアカウント ID は PlayerPrefs にキャッシュされます。ログアウト時やアカウント切り替え時は LoginManager.ClearLoginData() でクリアしてください。

環境構築の資料の1つです(VIVERSE 公式ドキュメントの和訳を再構成)。他の資料へは 資料一覧 から移動できます。
