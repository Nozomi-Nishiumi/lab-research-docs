テクスチャ欠如下での
3次元姿勢計測 実施要領

PART 02(追跡が困難な場合の代替案) で示した「キーポイント抽出が困難な場合の代替戦略」を、実際に着手できる手順レベルまで展開した別紙です。校正 → ベースライン → 主軸(DANNCE)→ CGモデル活用 → 検証、の順に、コマンド・設定・合否基準を記します。

方針の要約 ── 主軸は DANNCE/SDANNCE(ボリューメトリック回帰)。anipose 三角測量は ベースライン兼検証。CGモデルは ①合成学習データ生成 ②形状事前分布としての補助/相互検証。関節角の主計測はモデルバイアスを避け DANNCE 側に置く。
共通の注意 ── DANNCE/SDANNCE・aniposelib・PyTorch3D は更新が速い。以下のコマンド・パラメータ名は着手時点で各リポジトリの現行版を必ず確認すること。SDANNCE は Linux のみ検証。

SECTION 0

環境構築とディレクトリ規約

0-1手法ごとに環境を分ける

依存(CUDA・PyTorch・TensorFlow バージョン)が衝突するため、conda 環境を分離する。

bash
# 校正・三角測量ベースライン
conda create -n aniposelib python=3.9 -y
# DANNCE / SDANNCE（GPU必須・SDANNCEはLinux）
conda create -n dannce python=3.7 cudatoolkit=11.1 cudnn ffmpeg -c nvidia -y
# CGモデルのフィッティング（PyTorch3D）
conda create -n fit python=3.10 -y
合否基準各環境で nvidia-smi と python -c "import torch; print(torch.cuda.is_available())" が True を返す(DANNCE/fit 環境)。

0-2プロジェクトのディレクトリ規約

クラウド同期外(例 ~/Developer/frog-pose)に置き、git 管理。データは大容量なので .gitignore と DVC/外部ストレージを併用。

frog-pose/ ├── calib/ # 校正動画・calibration.toml ├── raw/session_YYYYMMDD/ # 同期済み本番動画（カメラ別） ├── anipose_project/ # A-3 ベースライン ├── dannce_project/ # A-4 主軸 ├── synth/ # B-2 合成データ・bpyスクリプト ├── fit/ # B-3 フィッティング └── validation/ # C ワンド・手動デジタイズ
SECTION A-1

撮影データと同期の規約

3台をハードウェアトリガで同期(PART 01)。全手法がフレーム完全同期を前提とする。カメラ名は全工程で一貫させ、ファイル名に埋め込む(例 ..._cam1.mp4)。DANNCE はカメラごとにサブフォルダを分け、動画名を先頭フレーム番号で命名する規約なので、撮影段階から命名を決めておく。

前提条件同期が甘いと高速な跳躍で三角測量・ボリューム構成が破綻する。トリガの実配線とタイムスタンプ一致を、本番前に LED 点滅で全視野同時に確認しておく。
SECTION A-2

カメラ校正(anipose / aniposelib)

A-2-1ChArUco ボードの準備

ChArUco を推奨(検出が頑健)。跳躍ボリュームと同スケールで、全視野に大きく映る板を用意。マス目寸法・マーカ寸法・辞書を控えておく(校正パラメータに使用)。

A-2-2aniposelib で校正(スクリプト方式)

bash
conda activate aniposelib
pip install aniposelib
python
from aniposelib.boards import CharucoBoard
from aniposelib.cameras import Camera, CameraGroup

# 板の実測値に合わせる（squaresX/Y, 単位mm）
board = CharucoBoard(7, 10,
                     square_length=25, marker_length=18.75,
                     marker_bits=4, dict_size=50)

cam_names = ['cam1', 'cam2', 'cam3']
cgroup = CameraGroup([Camera(name=n) for n in cam_names])

# カメラごとの校正動画（list of lists）
vidnames = [['calib/calib_cam1.mp4'],
            ['calib/calib_cam2.mp4'],
            ['calib/calib_cam3.mp4']]

# 反復バンドル調整で内部・外部を推定
cgroup.calibrate_videos(vidnames, board)
cgroup.dump('calib/calibration.toml')
合否基準平均再投影誤差が概ね 1px 未満。既知長ワンドで復元した点間距離の実長誤差(mm)を別途測り、記録として残す(検証 C で再使用)。誤差が大きい場合は誤検出フレームを除外して再校正。

A-2-3世界座標系を接地面・重力軸に合わせる

離陸角・重心の放物線を意味づけるため、床面や既知の直交治具で座標軸を定義する(anipose の [triangulation] axes / reference_point 相当)。重力方向を1軸に取ると後段のキネマティクスが解釈しやすい。

SECTION A-3

ベースライン:DLC/SLEAP + 三角測量

A-3-12D 検出器の学習

DeepLabCut または SLEAP でカエルのランドマーク(吻端・眼・肩・肘・手首・指端・股・膝・足首・指端・総排泄口・尾端など)をラベリングし学習。ここがテクスチャ欠如で崩れる想定なので、各部位の検出信頼度を後で診断できるよう記録する。

A-3-2anipose プロジェクトと config.toml

anipose は雛形を作らないので、config.toml を自分で用意する。骨長拘束(constraints)を入れると、テクスチャ欠如由来の外れを幾何拘束で多少緩和できる。

config.toml（抜粋）
model_folder = '/path/to/DLC_project'
nesting = 1
video_extension = 'mp4'

[calibration]
board_type = "charuco"
board_size = [7, 10]
board_marker_bits = 4
board_marker_dict_number = 50
board_marker_length = 18.75   # mm
board_square_side_length = 25  # mm

[triangulation]
cam_regex = '_cam([0-9])'
ransac = true
optim = true                # 3Dフィルタ有効
constraints = [["elbow","wrist"], ["knee","ankle"]]  # 骨長
scale_smooth = 2            # 時間平滑の強さ
scale_length = 2            # 骨長拘束の強さ
reproj_error_threshold = 3
score_threshold = 0.6

[angles]
knee = ["hip", "knee", "ankle"]
bash
conda activate aniposelib
anipose calibrate            # A-2 を config 方式で行う場合
anipose analyze             # DLCで2D検出
anipose filter              # 2Dフィルタ
anipose triangulate         # 3D復元 → pose-3d/*.csv
anipose calibration-errors  # 校正誤差の確認
anipose angles              # 関節角の算出
判定ゲート復元された部位のうち、再投影誤差・信頼度が慢性的に低い部位を列挙する。この診断が主軸を DANNCE へ切り替える根拠になる(全身が不良 → A-4 主軸へ / 一部のみ不良 → その部位を B のアンカー/補完対象に)。

A-4-2校正を DANNCE 形式へ変換

DANNCE は *dannce.mat に、カメラパラメータ(K・歪み RDistort/TDistort・回転 r・並進 t)、フレーム同期構造、(学習時は)3Dラベルをまとめて格納する。A-2 の calibration.toml からこの形式へ変換するスクリプトを用意する(VS Code の AI に雛形を書かせるのが早い)。

合否基準変換後のパラメータで既知点を再投影し、A-2 と同等の再投影誤差になること(座標系・単位・歪みモデルの取り違えがないことの確認)。

A-4-3ディレクトリ規約

dannce_project/ ├── videos/ │ ├── Camera1/0.mp4 # 先頭フレーム番号で命名 │ ├── Camera2/0.mp4 │ └── Camera3/0.mp4 ├── label3d_project_dannce.mat # 校正+同期(+3Dラベル) └── configs/*.yaml # com/dannce 設定

A-4-4COM(重心)ネットワーク

DANNCE は top-down 方式で、まず動物の重心を各視点で検出→三角測量して 3D COM を得る。COM を少数フレームでラベルし、COMfinder を学習・推論する。

bash
com-train   configs/com_config.yaml
com-predict configs/com_config.yaml

A-4-53Dキーポイントのラベリングと学習

Label3D で同期フレームに多視点ラベルを付け、三角測量で 3D 化。Label3D.exportDannce() で DANNCE 形式へ書き出す。まず数百フレーム規模から。跳躍のように四肢が大きく動く場合は、伸展位・屈曲位・回転を幅広くサンプルする。

dannce config.yaml（主要パラメータ・抜粋）
camnames: [Camera1, Camera2, Camera3]
n_channels_out: 14        # キーポイント数
vol_size: 120             # mm。最大伸展を包含する立方体の一辺
nvox: 80                  # voxel解像度（64/80…大きいほど精細・高メモリ）
batch_size: 1
predict_mode: torch
epochs: 100
最重要トレードオフvol_size(体積)を跳躍の最大伸展に合わせて大きく取る必要があるが、nvox を上げないと 1 voxel あたりの実寸が粗くなる。vol_size / nvox / VRAM の三すくみが本タスク最大の調整点。まず小さめ nvox で通し、精度を見て上げる。

A-4-6学習・推論

ラベルが少ない段階では、ラット由来の学習済み重み(RAT7M)からの転移学習が有効。

bash
# スクラッチ学習
dannce-train   configs/dannce_config.yaml
# もしくは事前学習からのfinetune
dannce-train   configs/dannce_finetune.yaml --train-mode finetune --epochs 100
# 推論 → predict_results/save_data_AVG0.mat
dannce-predict configs/dannce_config.yaml
合否基準hold-out ラベルとの Euclidean 誤差(mm)を部位別に集計。ワンド由来の実長誤差、再投影誤差、誤差の空間分布(ボリューム内均一性)を報告。SDANNCE の骨格拘束を入れると四肢末端の一貫性が上がるかを比較。
SECTION B-1

CGモデルのリグ確認・整備

購入した CG モデルを Blender で開き、アーマチュア(ボーン)とスキンウェイトの有無を確認する。無い/不十分な場合は、対象種の解剖に沿ってボーンを追加し、四肢が可動になるようウェイトを設定する(Rigify か手動)。この段階で関節の回転軸・可動域を、後段(合成データの姿勢サンプリング/フィッティングの拘束)で使う形に決めておく。

確認事項市販モデルは対象種の骨格ランドマークに較正されていない。関節角の絶対値を主計測にしない前提を崩さないこと(本手法は合成データ生成と相互検証が主用途)。
SECTION B-2

合成学習データ生成(bpy)

B-2-1bpy スクリプトの構成

Blender の Python API で、姿勢・照明・材質・背景をランダム化しつつ、実カメラと同じ内部/外部パラメータでレンダリングする。真値は自分が生成しているので、3D 関節座標を各視点へ投影して 2D/3D ラベルを自動出力できる。

python（bpy・擬似コード）
import bpy, random, json

for i in range(N):
    set_joint_angles(sample_within_biological_limits())  # B-1の可動域
    randomize_lighting()        # HDRI・強度・色
    randomize_material()        # テクスチャ依存を断つため見た目も多様化
    randomize_background()
    for cam in [cam1, cam2, cam3]:   # 実機のK・R・tを設定
        render(cam, f"synth/{cam.name}/{i:06d}.png")
        kp2d = project(joint_positions_3d, cam)  # 真値2D
        save_label(kp2d, joint_positions_3d)     # DLC/DANNCE形式

実写に寄せるため、センサノイズ・軽微な劣化を付与して sim-to-real ギャップを詰める。

B-2-2学習への投入

合成データで A-4(DANNCE)や A-3(DLC/SLEAP)を事前学習 → 少数の実ラベルで fine-tune。テクスチャの薄い対象ほど実写学習が難しいため、この底上げが効く。

合否基準合成のみで学習したモデルが、実写 hold-out で一定の精度を出すこと。出なければ、レンダリングの見た目分布(照明・材質・背景)を実機に寄せて再生成。
SECTION B-3

補助フィッティング(PyTorch3D)

B-3-1方針と参照実装

解析による合成(analysis-by-synthesis)を、微分可能レンダリングで実装する。動物向けの参照として SMALify(pose+shape+camera を 2Dキーポイント+シルエットに合わせて最適化)が有用だが、SMAL は四足哺乳類モデルでカエルには直接使えない。SMALify は方法論のテンプレートとして参照し、可動モデルは B-1 で整備したカエルメッシュ、または PyTorch3D 上で自前に構成する。

B-3-2PyTorch3D の構成

PyTorch3D の SoftSilhouetteShader でシルエットを微分可能にレンダリングし、観測マスクとの差を姿勢パラメータへ逆伝播する。カメラは 3 視点の校正済み R, T, K を PerspectiveCameras に設定。シルエット抽出は背景制御下の背景差分か SAM 等のセグメンテーション。

python（構成の骨子）
from pytorch3d.renderer import (
    PerspectiveCameras, RasterizationSettings,
    MeshRasterizer, MeshRenderer, SoftSilhouetteShader, BlendParams)

blend = BlendParams(sigma=1e-4, gamma=1e-4)
cams  = PerspectiveCameras(R=R3, T=T3, K=K3, device=dev)  # 3視点
rend  = MeshRenderer(MeshRasterizer(cams, RasterizationSettings(...)),
                     SoftSilhouetteShader(blend))

def loss(theta):                 # theta=全体回転・並進+関節角
    mesh = pose_mesh(theta)       # LBSで変形（B-1のリグ）
    sil  = rend(mesh)             # 3視点シルエット
    L  = w1*silhouette_iou(sil, obs_masks)
    L += w2*reproj(anchors, theta)   # 吻端・眼・総排泄口
    L += w3*temporal_smooth(theta)   # 関節角速度
    L += w4*joint_limit_prior(theta) # 可動域
    L += w5*shape_scale_prior(theta)
    return L

B-3-3最適化と初期化

  • 最適化:Adam で coarse-to-fine(まず全体の回転・並進・スケール → 次に四肢)。
  • 初期化:前フレーム+A-4(DANNCE)の 3D キーポイントで与え、局所解と左右対称の曖昧性を回避。3視点+陰影+アンカーが対称性を解消する。
  • 出力:関節角つき全身3D姿勢。ただし主計測は DANNCE 側に置き、これは相互検証/欠損部位の補完に用いる。
合否基準フィッティング解と DANNCE 3D の一致度(部位別の 3D 距離)。系統的な乖離があれば、それはモデル形態バイアスの兆候として methods に明記(主計測を DANNCE に置く根拠の裏づけになる)。
SECTION C

統合・検証プロトコル

手法に依らず、論文の妥当性を支える検証を用意する。

  • 既知長ワンド:跳躍ボリューム内で動かし、復元点間距離の実長誤差(mm)と誤差の空間分布を報告。
  • 手動デジタイズ比較:一部フレーム・一部部位で熟練者の手動 3D デジタイズと比較(観察者間誤差も)。
  • 重心の放物線を独立検証に使う:空中フェーズの重心は放物線を描くはず。再構成重心に放物線を当て、残差で計測ノイズを bound する。
  • 感度分析:欠損・オクルージョン処理や平滑化パラメータを振り、結論がそれらに頑健であることを示す。
循環論法の回避重心の弾道は検証専用とし、再構成側に弾道拘束を課さない。anipose の時間平滑は平滑化事前分布であって弾道拘束ではないが、平滑化が残差を過小評価しうる点は methods に一言添える。
SECTION D

意思決定ゲート

手戻りを避けるための分岐基準。

  • G1(A-3後):2D検出が全身で慢性的に不安定 → 主軸を A-4 DANNCE に確定。一部部位のみ不安定 → その部位を B のアンカー/補完対象に指定。
  • G2(A-4後):DANNCE でも特定部位(四肢末端等)が残る → SDANNCE の骨格拘束を試す、それでも残れば B-3 フィッティングで補完。
  • G3(ラベル律速):学習データ不足が精度を律速 → B-2 合成データで底上げしてから A-4/A-3 を再学習。
  • G4(検証):C の誤差が目標(要件定義の mm)を満たすまで、校正 → ボリューム/解像度 → ラベル量の順で改善を回す。
