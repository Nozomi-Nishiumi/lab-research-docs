PART 01

R を入れる

RStudio はエディタで、R 本体は含みません。先に R を入れてから RStudio を入れます(RStudio は入っている R を自動で認識します)。

macOS

Apple Silicon / Intel でビルドが異なる

  1. cran.r-project.org → Download R for macOS。
  2. チップに合うビルドを選ぶ。Apple メニュー → このMacについてで、「チップ」が M 系なら arm64、「プロセッサ」に Intel とあれば Intel 版。
  3. ダウンロードした .pkg を実行し、指示どおり進める。
  4. 3D可視化(別紙の rgl)などで必要になるので XQuartz も入れておく(推奨。入れたら Mac を再起動)。

Windows

base ディストリビューション

  1. cran.r-project.org → Download R for Windows → base。
  2. Download R for Windows のインストーラを取得。
  3. 実行し、オプションは既定のまま進める。
  4. 完了。R のアイコンが出るが、以降は R を直接ではなく RStudio から使う。
PART 02

RStudio を入れる

RStudio Desktop(無料)

Posit 提供・オープンソース版は完全無料

  1. posit.co/download/rstudio-desktop を開く。ページが OS を判定し、適切なインストーラを案内する。
  2. macOS:.dmg を開き、RStudio を「アプリケーション」へドラッグ。Windows:.exe を実行し既定のまま進める。
  3. RStudio を起動。インストール済みの R を自動検出して、左下に R のバージョンが表示されれば成功。
補足 ── すでに VS Code を使っているなら、Posit の Positron(VS Code ベースで R と Python を1つで扱える IDE)も選択肢です。ただし R の学習・チュートリアルはほぼ RStudio 前提なので、まずは RStudio が無難です。

共通の初期設定

研究用途ならこの3つは最初に

  1. ワークスペースを毎回クリーンに ── Tools → Global Options → General で「Save workspace to .RData on exit」を Never に、「Restore .RData into workspace at startup」のチェックを外す。前回の変数が勝手に残らなくなり、スクリプトだけで結果を再現できるようになります(再現性の基本設定)。
  2. 解析はプロジェクト単位で ── File → New Project… でプロジェクトを作り、データ・スクリプトをその中に置く。作業ディレクトリが自動で切り替わり、パスのトラブルが減ります。
  3. 見た目の調整(任意) ── Tools → Global Options → Appearance でエディタテーマとフォントサイズを変更できます。
ヒント ── プロジェクトは ~/Developer のようなクラウド同期外のフォルダに置くと、解析の中間ファイルの同期衝突を避けられます(VS Code 資料と同じ方針)。
PART 03

パッケージと動作確認

可視化には ggplot2 を使います。tidyverse(ggplot2・dplyr などの詰め合わせ)をまとめて入れるのが簡単です。RStudio 左下の Console に次を貼って実行(Enter)。初回は1〜3分ほどかかります。

R · Console
# 初回のみ（インストール）
install.packages("tidyverse")

# 動作確認
library(ggplot2)
ggplot(mpg, aes(displ, hwy)) + geom_point()
つまずき対策 ── ダウンロードで止まる場合は CRAN ミラーを変更:options(repos = "https://cloud.r-project.org/") を実行してから再度インストール。権限エラーが出たら、個人ライブラリ作成の確認に「yes」で進めて問題ありません。
PART 04

AI と連携させる

RStudio には2系統の AI 連携があります。この2つは役割が違うので、どちらか一方ではなく両方設定して使い分けるのがおすすめです(VS Code 資料の Copilot / Cline と同じ関係)。GitHub Copilot はタイプ中のインライン補完(書く速度を上げる)、Posit Assistant は R セッション・データ・プロットを見て動くエージェント(作業を代行させる)。Copilot は GitHub アカウントでのサインインが必要、Posit Assistant はモデルの API キーがあれば動きます。

01
GitHub Copilot補完タイプ・VS Code と同じ Copilot
学生・教員は無料 / 他は要サブスク
  1. RStudio が 2023.09.0 以降か確認(Help → Check for Updates)。古い場合は更新する。
  2. Tools → Global Options → Assistant(版により Copilot)を開き、「Use code assistant」で GitHub Copilot を選ぶ。
  3. 案内に従って Copilot Agent のコンポーネントをインストール(初回のみ。数十秒)。
  4. Sign In を押すとダイアログに verification code(8桁)が出る。ブラウザで github.com/login/device を開き、コードを貼って Authorize GitHub Copilot Plugin。
  5. ダイアログに「Copilot: Ready」等のステータスが出たら閉じる。GitHub 側でサブスクが有効でないと Sign In 後にエラーになるので、その場合は次の note を確認。
  6. .R ファイルを開いてコードを書き始めると、続きが薄いゴーストテキストで提案される。Tab で確定、無視してタイプを続ければ消える。コメントで # 気温のヒストグラムを描く と書いて改行すると、コードを丸ごと提案してくることも確認してみる。
要件と無料枠 ── GitHub アカウントと Copilot のサブスク(Pro / Business)が必要。学生・教員は GitHub Education の申請で Pro が無料(学生証か大学メールで認証、承認まで数日)。まず Education を申請してから設定するのが定石。詳細は RStudio の Copilot ドキュメント。
02
Posit Assistantチャット/エージェント・データを見て動く
モデルは持ち込み(BYOK)

2026年に RStudio へ搭載された新しいデータサイエンス向けエージェント。R セッションを見られるのが強みで、コンソール出力・プロット・環境内の変数を踏まえて、探索的分析・データ整形・モデリングを手伝います。別途コンテキストを渡さなくても、あなたのデータや図を「見た」うえで動きます。

  1. 最近の RStudio に同梱(Help → Check for Updates で最新化)。サイドバーの Posit Assistant(チャット)ペインを開く。
  2. モデルプロバイダを選ぶ:Anthropic(Claude)/ GitHub Copilot / OpenAI / Posit AI。Copilot を上で設定済みならそのまま Copilot サインインが手早い。Claude を使うなら A の手順で API キーを用意する。
  3. A) Anthropic(Claude)の API キーを使う

    ① キーを取得:console.anthropic.com でアカウントを作成 → API Keys → Create Key → 生成されたキー(sk-ant-… で始まる)をコピー。表示は一度きりなので確実に控える。従量課金なので少額(例:$5)のクレジットを入れておく。

    ② Assistant に設定:Posit Assistant ペインの設定でプロバイダに Anthropic を選び、①のキーを貼り付け。

    ③ モデルを選ぶ:標準は Claude Sonnet 系(能力と価格のバランス)。軽い用途なら Haiku 系(高速・安価)。

  4. 動作確認:data(airquality) を読み込んだ状態で、日本語で「airquality の Ozone の分布を可視化して」と指示。コード生成 → 実行提案 → Plots ペインに図、まで流れれば成功。
  5. 以後は、データを読み込んだ状態で「この分布を可視化して」「この2群を検定して」のように指示する。コード実行・ファイル編集は承認制。
BYOK(持ち込み) ── モデル利用料は各プロバイダに直接支払い(例:Anthropic の API キー、Copilot のサブスク)。API キーは他人に見せず、スクリプトやリポジトリに貼らないこと。有効化手順は版で変わるため、着手時に Posit のドキュメントで現行版を確認のこと。

使い分けと切り替え

2つは同時に動く。排他ではなく共存

Copilot はエディタの中で動き(タイプすると薄いゴーストテキストで続きが出る)、Posit Assistant は専用のチャットペインで動きます。互いに干渉しないので、「切り替え」というより今どちらの手を使うかを選ぶだけです。

  1. Copilot を使う ── スクリプトを普通に編集し、出てきた候補を Tab で確定。常時オンの背景アシスト。
  2. Posit Assistant を使う ── Assistant ペインを開き、やりたいことを日本語で書いて実行。データを読み込んだ状態で頼むと精度が上がる。コード実行は承認制。
  3. 補完が邪魔なとき ── Tools → Global Options → Assistant で Copilot を一時的にオフにできる(設定はいつでも戻せる)。

Copilot が向く場面

  • 自分で書きながら、続きをサッと補完したい
  • ggplot の定型句・データ整形の1ステップ
  • タイピングの流れを止めたくない
  • 関数名・引数をうろ覚えのときの補助

Posit Assistant が向く場面

  • 読み込んだデータを見せて探索的分析を任せたい
  • 「この図を作って」「この検定をして」の丸投げ
  • エラーメッセージの原因調査と修正
  • 解析スクリプト全体の雛形作成
判断の目安 ── 「自分で書く隣に Copilot、まるごと任せるなら Posit Assistant」。VS Code 資料の「Copilot と Cline」と同じ関係です。迷ったら、書き始めは Copilot、詰まって「全体を任せたい」と思ったら Assistant に渡す、で回せます。
