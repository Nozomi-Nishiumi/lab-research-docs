PART 01

仕組み — すべてを1つのループで

3つの関数だけ

OpenCV が取り込みもウィンドウ表示も面倒を見てくれる

  • cap.read() ── カメラから1フレーム取り込む。次のフレームが来るまでブロックする。
  • imshow() ── フレームをウィンドウに表示する(ウィンドウ生成も込み)。
  • waitKey(1) ── キー入力を1ms待つ。この呼び出しがウィンドウの描画処理も担っているので、書き忘れると画面が更新されない(初心者が最初に踏む罠)。
まずはこれで十分 ── カメラの動作確認、露出や画角の調整、簡単な画像処理の試作なら、この単一ループ構成が最短です。限界が見えたら(PART 04)、次のページへ。

Python で動かす

opencv-python ひとつだけ

venv の作成と Interpreter 選択は済んでいる前提(前提の確認)

  1. 作業フォルダ(例:opencv-camera)を File → Open Folder で VS Code で開く。venv 未作成のフォルダなら、VS Code 導入の PART 03 の手順で venv を作って Interpreter を選んでおく。
  2. 統合ターミナル ⌃` / Ctrl+` を開き、プロンプト先頭に (.venv) が付いていることを確認(付いていなければターミナルを開き直す)。

    VS Code ターミナル
    pip install opencv-python
  4. capture_min.py をダウンロードし、VS Code のエクスプローラー(左のファイル一覧)へドラッグ&ドロップして作業フォルダに置く。

    VS Code ターミナル
    python capture_min.py
  6. カメラ映像のウィンドウが開き、左上に実測 fps が出れば成功。ESC で終了。カメラを選ぶなら python capture_min.py 1 のように番号を渡す。
エディタ右上の ▷ でも実行できる ── capture_min.py を開いて「Run Python File」でも動きます(カメラ番号の引数を渡すときはターミナルから)。
macOS のカメラ許可 ── 初回に「Visual Studio Code がカメラにアクセスしようとしています」の確認が出るので許可する。出ないまま開けない場合は システム設定 → プライバシーとセキュリティ → カメラ で VS Code を ON にする。
Python · capture_min.py(全文)
#!/usr/bin/env python3
# ============================================================
# capture_min.py — OpenCV だけでカメラを映す最小構成
#
# 取り込み(cap.read)と表示(imshow)を1つのループで行う、
# いちばん素朴な形。まずこれでカメラと OpenCV の動作を確認する。
#
# 使い方:
#   python capture_min.py            # カメラ 0 を開く
#   python capture_min.py 1          # カメラ番号を指定
#   ESC で終了
#
# 必要: pip install opencv-python
# ============================================================

import sys
import time
import cv2

camera_id = int(sys.argv[1]) if len(sys.argv) > 1 else 0

cap = cv2.VideoCapture(camera_id)
if not cap.isOpened():
    raise SystemExit(f"カメラ {camera_id} を開けません")

# 解像度の要求(カメラが対応していなければ別の値になる)
cap.set(cv2.CAP_PROP_FRAME_WIDTH, 1280)
cap.set(cv2.CAP_PROP_FRAME_HEIGHT, 720)
print("解像度:", int(cap.get(cv2.CAP_PROP_FRAME_WIDTH)), "x",
      int(cap.get(cv2.CAP_PROP_FRAME_HEIGHT)))

n, fps, t0 = 0, 0.0, time.perf_counter()
while True:
    ok, frame = cap.read()          # 1フレーム取り込む(ブロッキング)
    if not ok:
        break

    # 実測fpsを左上に描く(0.5秒ごとに更新)
    n += 1
    dt = time.perf_counter() - t0
    if dt >= 0.5:
        fps = n / dt
        n, t0 = 0, time.perf_counter()
    cv2.putText(frame, f"{fps:5.1f} fps", (20, 40),
                cv2.FONT_HERSHEY_SIMPLEX, 1.0, (0, 255, 0), 2)

    cv2.imshow("camera", frame)     # 表示(ウィンドウは OpenCV 任せ)
    if cv2.waitKey(1) & 0xFF == 27:  # ESC で終了。waitKey が無いと描画されない
        break

cap.release()
cv2.destroyAllWindows()

  3. 以降のコマンドは統合ターミナル ⌃` / Ctrl+` で、OS に応じて下のどちらかの列を実行する。

macOS

Homebrew + 1行コンパイル(GLFW は不要)

  1. OpenCV を入れる。
    VS Code ターミナル
    brew install opencv pkg-config
  2. コンパイルする。
    VS Code ターミナル · コンパイル
    clang++ -std=c++17 -O2 capture_min.cpp \
  -o capture_min \
  $(pkg-config --cflags --libs opencv4)
  3. 実行:./capture_min。ESC で終了。

Windows

vcpkg + CMake(GLFW は不要)

  1. vcpkg で OpenCV を入れる(初回は数十分)。
    VS Code ターミナル(PowerShell)
    git clone https://github.com/microsoft/vcpkg
.\vcpkg\bootstrap-vcpkg.bat
.\vcpkg\vcpkg install opencv4
  2. CMakeLists.txt を同じフォルダに置いてビルド(フォルダにあるソースだけをビルドする作りになっている)。
    VS Code ターミナル(PowerShell)· ビルド
    cmake -B build ^
  -DCMAKE_TOOLCHAIN_FILE=.\vcpkg\scripts\buildsystems\vcpkg.cmake
cmake --build build --config Release
  3. 実行:.\build\Release\capture_min.exe。ESC で終了。

capture_min.cpp(全文)

C++ · capture_min.cpp
// ============================================================
// capture_min.cpp — OpenCV だけでカメラを映す最小構成
//
// 取り込み(cap.read)と表示(imshow)を1つのループで行う、
// いちばん素朴な形。まずこれでカメラと OpenCV の動作を確認する。
//
// ビルド:
//   macOS : brew install opencv pkg-config
//           clang++ -std=c++17 -O2 capture_min.cpp -o capture_min \
//             $(pkg-config --cflags --libs opencv4)
//   Windows: 同梱の CMakeLists.txt + vcpkg を使用(解説ページ参照)
//
// 使い方:
//   ./capture_min            # カメラ 0 を開く
//   ./capture_min 1          # カメラ番号を指定
//   ESC で終了
// ============================================================

#include <opencv2/opencv.hpp>
#include <chrono>
#include <cstdio>

int main(int argc, char** argv) {
    const int cameraId = (argc > 1) ? std::atoi(argv[1]) : 0;

    cv::VideoCapture cap(cameraId);
    if (!cap.isOpened()) {
        std::fprintf(stderr, "カメラ %d を開けません\n", cameraId);
        return 1;
    }

    // 解像度の要求(カメラが対応していなければ別の値になる)
    cap.set(cv::CAP_PROP_FRAME_WIDTH, 1280);
    cap.set(cv::CAP_PROP_FRAME_HEIGHT, 720);
    std::printf("解像度: %d x %d\n",
                int(cap.get(cv::CAP_PROP_FRAME_WIDTH)),
                int(cap.get(cv::CAP_PROP_FRAME_HEIGHT)));

    cv::Mat frame;
    int n = 0;
    double fps = 0.0;
    auto t0 = std::chrono::steady_clock::now();

    while (cap.read(frame)) {       // 1フレーム取り込む(ブロッキング)
        // 実測fpsを左上に描く(0.5秒ごとに更新)
        ++n;
        const double dt = std::chrono::duration<double>(
            std::chrono::steady_clock::now() - t0).count();
        if (dt >= 0.5) {
            fps = n / dt;
            n = 0;
            t0 = std::chrono::steady_clock::now();
        }
        char buf[32];
        std::snprintf(buf, sizeof(buf), "%5.1f fps", fps);
        cv::putText(frame, buf, {20, 40}, cv::FONT_HERSHEY_SIMPLEX,
                    1.0, {0, 255, 0}, 2);

        cv::imshow("camera", frame);   // 表示(ウィンドウは OpenCV 任せ)
        if (cv::waitKey(1) == 27) break;  // ESC で終了。waitKey が無いと描画されない
    }
    return 0;
}
PART 04

この方式の限界 — 次のページへ

単一ループの天井

動くようになったら、fps 表示を見ながら考える

  • 取り込みと表示が直列 ── read → imshow → waitKey を順番に回すため、表示にかかる時間のぶんだけ取り込みが待たされる。カメラが120fps出せても、ループ全体が間に合わなければその速度では取り込めない。
  • waitKey が不安定要素 ── waitKey(1) は「最低1ms」待つだけで、実際は GUI 処理を含めて数ms かかることもある。フレームレートの上限と揺らぎがここで決まってしまう。
  • 処理を足すほど遅くなる ── 解析や録画をこのループに足すと、その時間もまるごと取り込み間隔に上乗せされ、フレームを取りこぼす。
つまり ── 「確実に高速で取り込みたい」なら、取り込みを専用スレッドに分離し、表示は別ループに任せる構成が必要になります。それが次のページの主題です。
